# Product Requirements — Chronicle

Version: 0.1
Date: 2026-06-14

Chronicle is an AI Biographer: a long-term witness that helps a person discover meaning, patterns, contradictions, and legacy from their lifetime of memories and artifacts. Chronicle combines Subject Memory (facts, artifacts, timeline) and Biographer Memory (interpretations, observations, hypotheses) to surface narratives the user can curate.

Goals & outcomes
- Discover recurring themes and life patterns for a user.
- Surface contradictions and help unpack shifting beliefs.
- Preserve voice and produce legacy artifacts (memoirs, manifestos, letters).
- Guide reflective interviews that fill gaps and deepen meaning.

Target audience
- Individuals documenting a life (founders, creatives, elders).
- Families preserving legacy.
- Professional biographers and historians.

Core principles
- Push Back: AI should challenge and probe, not only validate.
- Themes Over Events: biography emerges from patterns and themes.
- Preserve Original Voice: assist without overwriting the user's voice.
- Ask Better Questions: prioritize interviews that close meaningful gaps.
- Build Legacy: produce well-cited, exportable artifacts.

Major capabilities (product framing)
- Capture: import or record multi-year artifacts (docs, photos, audio, social exports).
- Memory: canonicalize facts into Subject Memory and derive interpretable Biographer Memory.
- Discovery: surface themes, timeline, and contradictions transparently with evidence.
- Interview: run adaptive, prioritized interviews to resolve open questions.
- Compose: assist users in turning themes into chapters and book drafts with provenance and voice preservation.

Deliverables covered by this document set
- Product vision and prioritized feature set
- Domain model and user-facing behaviors
- Database design summary and migration guidance
- API surface for ingestion, memory retrieval, interviews, and book generation
- UX concepts and writing studio flows
- MVP and roadmap milestones
